Acacia

Accessibility

Acacia provides a clear PDF workflow for macOS, iPhone, and iPad with labelled controls, readable layouts, keyboard-reachable actions, and support for Apple accessibility settings across common library, reading, annotation, search, export, and account tasks.

Supported Features

  • VoiceOver Primary controls, tabs, document rows, reader actions, annotations, page controls, and status updates use accessible names, roles, hints, values, and actions.
  • Voice Control Controls expose clear labels such as Open PDF, Search documents, Highlight tool, Next page, Export, and Restore purchases so they can be targeted with voice commands.
  • Larger Text Text inputs and touch targets support larger text settings, including 200% sizing for key search, navigation, page, signature, and annotation controls.
  • Dark Interface The app follows the system dark appearance for common tasks, including the library, document viewer, mobile controls, sheets, and annotation surfaces.
  • Differentiate Without Color Alone Selection state is conveyed with labels, icons, borders, check states, page counters, and text, not only color.
  • Sufficient Contrast Text, controls, and selected states use strong foreground and border contrast, and honor Increased Contrast and Darker System Colors where available.
  • Reduced Motion Press animations and transition effects are reduced or disabled when Reduce Motion or Prefer Cross-Fade Transitions is enabled.
  • Captions Acacia's core PDF workflow does not require audio or video. Any app-provided instructional or promotional video is prepared to use captions or synchronized text alternatives.
  • Audio Descriptions Acacia's core tasks are fully visual/textual and do not depend on video-only instructions. Any future time-based visual media will include an audio description or equivalent text alternative.

Feedback

If you encounter an accessibility issue, email support@benebsworth.com with the workflow, macOS version, and assistive technology you are using.
